Recuperación de Contraseña
Flujo completo de restablecimiento seguro de acceso
📅 Fecha de implementación: Marzo 2026

⚙️ Tecnología: Supabase Auth + Node.js/Express

1. Resumen Ejecutivo

¿Qué se implementó?

  • ✅ Un flujo completo de recuperación de contraseña para todos los usuarios del sistema (doctores, secretarias y administradores).
  • ✅ El sistema envía un email automático con un enlace seguro y de uso único al correo registrado del usuario.
  • ✅ El usuario puede establecer una nueva contraseña de forma segura sin intervención del administrador.
  • ✅ El enlace de recuperación expira automáticamente por seguridad (tiempo configurado en Supabase, por defecto 1 hora).
  • ✅ La funcionalidad se integra nativamente con Supabase Auth, que es la única fuente de verdad de autenticación del sistema.

Anteriormente, si un usuario olvidaba su contraseña, requería la intervención manual de un administrador para restablecerla. Con esta implementación, el proceso es completamente autónomo, seguro y auditado por Supabase.

2. Ciclo Completo del Proceso

A continuación se describe cada paso del flujo desde que el usuario olvida su contraseña hasta que accede nuevamente al sistema:

1
👤 Usuario

Hace click en "¿Olvidaste tu contraseña?"

En la pantalla de inicio de sesión, el usuario hace click en el enlace. El sistema lo redirige a la página de recuperación de contraseña (/forgot-password).

2
👤 Usuario

Ingresa su correo electrónico

El usuario escribe el email con el que está registrado en el sistema y presiona "Enviar enlace de recuperación".

3
⚙️ Sistema

El servidor procesa la solicitud

El backend recibe el email y llama a la API de Supabase (auth.resetPasswordForEmail()). Supabase genera un token único y seguro para ese usuario.

4
📧 Email

Supabase envía el email de recuperación

De forma automática, Supabase envía un correo al email ingresado con un botón/enlace único. Este enlace tiene una validez limitada y solo puede ser usado una vez. Si el email no existe en el sistema, no se muestra ningún error (por seguridad).

5
👤 Usuario

El usuario abre el email y hace click en el enlace

El usuario revisa su bandeja de entrada y hace click en el botón "Restablecer contraseña". El navegador lo lleva a la página /reset-password del sistema con un token de seguridad en la URL.

6
👤 Usuario

Ingresa su nueva contraseña

La página detecta automáticamente el token en la URL. El usuario escribe su nueva contraseña (mínimo 6 caracteres) y la confirma. El sistema valida que ambas coincidan.

7
⚙️ Sistema

El sistema actualiza la contraseña en Supabase

El backend verifica el token, extrae la identidad del usuario y actualiza la contraseña en Supabase usando la API administrativa (auth.admin.updateUserById()). La contraseña se almacena siempre de forma cifrada (bcrypt).

8
👤 Usuario

Redirección al inicio de sesión

Se muestra un mensaje de éxito y el sistema redirige automáticamente a la pantalla de login. El usuario puede ingresar inmediatamente con su nueva contraseña.

⚠️

Configuración requerida en Supabase: Para que el envío de emails funcione correctamente en producción, se debe configurar un servidor SMTP en el panel de Supabase (Authentication → Settings → SMTP). En el plan gratuito de Supabase, el envío de emails está limitado a 3 emails por hora. Para un entorno productivo se recomienda configurar un proveedor como SendGrid, Resend o AWS SES.

5. Medidas de Seguridad

La implementación incorpora múltiples capas de seguridad para proteger el proceso de recuperación:

🎟️

Token de Uso Único

El enlace de recuperación contiene un token criptográfico único que se invalida al ser utilizado o al expirar.

⏱️

Expiración Automática

El token expira automáticamente (1 hora por defecto). Pasado ese tiempo, el enlace deja de funcionar.

🤐

Sin Exposición de Datos

El sistema no revela si un email existe o no. La respuesta es siempre la misma para prevenir enumeración.

🔒

Contraseñas Cifradas

Supabase almacena todas las contraseñas usando bcrypt. Nunca se guarda texto plano en la base de datos.

🛡️

Admin API

El cambio de contraseña se ejecuta con la clave de servicio de Supabase, protegida exclusivamente en el backend.

✅

Validación de Formulario

El frontend valida longitud mínima (6 caracteres) y coincidencia de contraseñas antes de enviar la solicitud.

7. Endpoints de la API
Método Endpoint Descripción Requiere Auth
POST /api/auth/forgot-password Envía email de recuperación al correo indicado No
POST /api/auth/reset-password Actualiza la contraseña usando el token del email No (usa token)
GET /forgot-password Renderiza la página de solicitud de recuperación No
GET /reset-password Renderiza la página de nueva contraseña No
ℹ️

Nota técnica: El token de recuperación que Supabase incluye en la URL del email tiene el formato /reset-password#access_token=TOKEN&type=recovery. La página lo extrae automáticamente desde el hash de la URL usando JavaScript del lado del cliente, sin que el token viaje nunca al servidor hasta que el usuario confirma la nueva contraseña.
